Encryption in transit is a baseline requirement for almost all Kubernetes environments today, and forms the foundation of a zero-trust security posture. However, the challenge with security is that it doesn’t come without a cost: it often involves a trade-off between complexity, user experience, and performance. While most Cloud Native users will know of Istio as a service mesh, providing advanced HTTP functionality, it can also serve the role of providing a foundational network security layer.
